Module 2: The Development of Cybersecurity

Cybersecurity has evolved significantly over time.
As technology grows, threats become more advanced, requiring better strategies and understanding.

This module explores the history of cybersecurity, types of attacks, and key security domains.

1. Overview of Cybersecurity Development​

Cybersecurity is constantly evolving.
Modern cyberattacks are often improved versions of older attacks.

👉 Understanding the past helps us:

  • Learn how attacks work
  • Build better defense strategies

2. History of Cyberattacks​

Early Attacks​

  • Brain Virus (1986)
    One of the first computer viruses

  • Morris Worm (1988)
    One of the first widespread internet worms

Modern Attacks​

  • ILOVEYOU Virus (2000)
    Spread via email and caused massive damage

  • Equifax Data Breach (2017)
    Exposed sensitive data of millions of users

👉 Lesson:
Cyber threats evolve, but many techniques remain similar.

3. Main Types of Cyber Threats​

1. Malware​

Malware is software designed to harm systems.

Types include:

  • Virus
  • Worm
  • Ransomware
  • Spyware

2. Phishing Attacks​

Phishing tricks users into giving sensitive information.

Common types:

  • Spear phishing (targeted attack)
  • Whaling (targeting executives)
  • BEC (Business Email Compromise)
  • Vishing (voice phishing)
  • Smishing (SMS phishing)

3. Social Engineering​

Instead of attacking systems, attackers manipulate people.

Examples:

  • Social media phishing
  • Watering hole attack
  • USB baiting
  • Physical social engineering

👉 Key idea:
Hackers often attack humans, not computers.

4. Identifying Attack Methods​

Understanding how attacks work helps us defend against them.

Common attack methods:​

  • Malware attacks → damage systems
  • Phishing attacks → steal information
  • Social engineering → manipulate people

Key principle:​

Don’t just protect systems — protect users as well.

5. Types of Cyber Attacks​

  • Password attacks → guessing or cracking passwords
  • Social engineering → deception techniques
  • Physical attacks → targeting devices or environments
  • Adversarial AI → exploiting AI systems
  • Supply chain attacks → targeting third-party vendors
  • Cryptographic attacks → breaking encryption

6. Understanding the Attacker​

A threat actor is any individual or group that poses a security risk.

Types of attackers:​

  • APT (Advanced Persistent Threat)

    • Highly skilled
    • Long-term attacks
    • Hard to detect
    • Insider threats
    • Internal users abusing access

  • Hacktivists

    • Politically or socially motivated

Types of hackers:​

  • White hat → ethical hackers
  • Gray hat → sometimes legal, sometimes not
  • Black hat → malicious attackers

7. CISSP – 8 Security Areas​

Cybersecurity can be divided into 8 main domains:

  1. Security and Risk Management

    • Managing risks and policies

  2. Asset Security

    • Protecting data and resources

  3. Security Architecture and Engineering

    • Designing secure systems

  4. Communication and Network Security

    • Protecting data in transit

  5. Identity and Access Management (IAM)

    • Controlling user access

  6. Security Assessment and Testing

    • Evaluating system security

  7. Security Operations

    • Monitoring and responding to threats

  8. Software Development Security

    • Building secure applications

8. Key Terminology​

  • Malware: Harmful software
  • Phishing: Trick users to reveal data
  • Social Engineering: Manipulating people
  • Threat Actor: Attacker
  • APT: Advanced long-term attack
  • CIA Triad: Confidentiality, Integrity, Availability
  • CISSP: Security certification framework

Conclusion​

Cybersecurity is constantly evolving alongside technology.

To stay secure, we must:

  • Understand past attacks
  • Recognize attack methods
  • Identify attacker behavior
  • Apply strong security principles

👉 Final takeaway:

The better you understand attacks, the better you can defend against them.
